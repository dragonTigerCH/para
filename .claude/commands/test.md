---

테스트 스위트를 실행하세요:

1. 프로젝트의 모든 테스트 실행 (긴 실행 시간 테스트 제외)
2. 테스트 결과를 명확하게 보고:
   - 통과한 테스트 수
   - 실패한 테스트 수
   - 실패 시 세부 정보
3. 테스트가 실패하면 TDD의 어느 단계에 있는지 식별:
   - Red: 새 테스트에 대한 예상된 실패
   - Green: 예상치 못한 실패 - 수정 필요
   - Refactor: 의도치 않게 동작이 변경됨

기억하세요:
- 커밋하기 전에 모든 테스트가 통과해야 함
- 모든 구조적 변경 후 테스트 실행
- 테스트를 통과시키는 코드 구현 후 테스트 실행
