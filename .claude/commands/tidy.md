description: 동작을 변경하지 않고 구조적 변경(리팩토링)만 수행 - Tidy First 접근법
---

구조적 변경을 위한 Tidy First 접근법을 따르세요:

1. 현재 모든 테스트가 통과하는지 확인 (Green 상태)
2. 필요한 구조적 개선사항 식별:
   - 명확성을 위한 변수, 메서드 또는 클래스 이름 변경
   - 복잡성을 줄이기 위한 메서드 추출
   - 코드를 더 나은 위치로 이동
   - 중복 제거
   - 코드 구성 개선
3. 한 번에 하나의 구조적 변경만 수행
4. 각 변경 후 모든 테스트를 실행하여 동작이 변경되지 않았는지 확인
5. 구조적 변경을 동작 변경과 별도로 커밋

기억하세요:
- 구조적 변경과 동작 변경을 절대 섞지 마세요
- 테스트가 통과할 때만 리팩토링
- 테스트를 실행하여 동작 변경이 없는지 검증
- 확립된 리팩토링 패턴 사용
- 변경사항을 작고 집중적으로 유지
