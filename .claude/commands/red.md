description: RED 단계 - 작은 단위의 기능을 정의하는 실패하는 테스트 작성
---

TDD의 RED 단계를 실행하세요:

1. 다음으로 구현할 작은 기능 단위를 식별
2. 실패하는 테스트를 작성하되:
   - 동작을 설명하는 의미 있는 이름 사용 (예: "두개의양수를더해야함")
   - 하나의 특정 동작만 테스트
   - 가능한 한 단순하게
   - 명확한 실패 메시지 포함
3. 테스트를 실행하여 올바른 이유로 실패하는지 확인
4. 아직 프로덕션 코드를 구현하지 마세요

기억하세요:
- 실패할 수 있는 가장 단순한 테스트 작성
- 구현이 아닌 동작을 테스트
- 한 번에 하나의 테스트만
- GREEN 단계로 진행하기 전에 테스트가 실패하는지 확인
