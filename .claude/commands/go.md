다음 테스트에 대한 TDD 사이클(Red → Green → Refactor)을 따르세요:

1. plan.md를 읽고 다음 미완료 테스트를 찾기
2. 실패하는 테스트 작성 (Red 단계)
3. 모든 테스트를 실행하여 실패하는지 확인
4. 테스트를 통과시키는 데 필요한 최소한의 코드 구현 (Green 단계)
5. 모든 테스트를 실행하여 통과하는지 확인
6. 필요하면 리팩토링 고려 (Refactor 단계)
7. plan.md에서 테스트를 완료로 표시

기억하세요:
- 테스트를 통과시키는 데 충분한 코드만 작성
- 변경사항을 작고 집중적으로 유지
- 구조적 변경과 동작 변경을 분리
- 모든 테스트가 통과할 때만 커밋
