description: REFACTOR 단계 - 모든 테스트를 통과시키면서 코드 구조 개선
---

TDD의 REFACTOR 단계를 실행하세요:

1. 모든 테스트가 통과하는지 확인 (GREEN 상태여야 함)
2. 필요한 개선사항 식별:
   - 중복 제거
   - 이름과 명확성 개선
   - 더 나은 구성을 위해 메서드 추출
   - 복잡한 로직 단순화
3. 한 번에 하나의 리팩토링 변경만 수행
4. 각 리팩토링 단계 후 모든 테스트 실행
5. 테스트가 실패하면 되돌리고 다른 접근법 시도
6. 코드가 깨끗해질 때까지 계속

일반적인 리팩토링 패턴:
- 메서드 추출
- 변수/메서드 이름 변경
- 중복 제거
- 조건부 로직 단순화
- 상수 추출

기억하세요:
- 테스트가 GREEN일 때만 리팩토링
- 한 번에 하나의 리팩토링만
- 각 변경 후 테스트 실행
- 동작이 변경되어서는 안 됨
- 구조적 변경을 별도로 커밋
